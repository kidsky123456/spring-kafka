# Kafka 多Topic 双向通信设计（SpringBoot3）
先直接回答你的核心疑问：
**第二套项目必须同时有【消费者 + 生产者】**
- 消费者：监听第一套项目发送的原始消息（Topic A）
- 生产者：处理完消息后，把结果发送给第一套项目（Topic B）

如果第二套只有生产者，**根本收不到第一套的消息**，无法完成业务处理。

---

## 整体架构设计（同一套Kafka，两个Topic）
### 核心流程
1. **Topic 命名**
   - `topic-request`：项目一 → 项目二（原始请求消息）
   - `topic-response`：项目二 → 项目一（处理结果消息）
2. **项目分工**
   - **项目一（服务端A）**：
     ✅ 生产者：发送消息到 `topic-request`
     ✅ 消费者：监听 `topic-response` 接收处理结果
   - **项目二（服务端B）**：
     ✅ 消费者：监听 `topic-request` 接收原始消息
     ✅ 生产者：处理后发送结果到 `topic-response`

### 架构图
```
项目A(生产者) → topic-request → 项目B(消费者)
项目B(生产者) → topic-response → 项目A(消费者)
```

---

## 环境准备
1. SpringBoot 3.2.x
2. Kafka 3.x（本地/远程同一套集群）
3. JDK 17+
4. 依赖：`spring-kafka`

# 第一套项目：项目A（生产者 + 消费者）
## 1. application.yml 配置
```yaml
spring:
  kafka:
    # 同一套Kafka地址
    bootstrap-servers: localhost:9092
    # 生产者配置
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      retries: 3
    # 消费者配置
    consumer:
      group-id: group-A
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: latest

# 自定义Topic名称
kafka:
  topic:
    request: topic-request
    response: topic-response
```

## 2. 生产者：发送消息到 topic-request
```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

/**
 * 项目A生产者：发送请求消息给项目B
 */
@Service
public class ProjectAProducer {

    @Value("${kafka.topic.request}")
    private String requestTopic;

    private final KafkaTemplate<String, String> kafkaTemplate;

    // 构造器注入（SpringBoot3推荐）
    public ProjectAProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    /**
     * 发送业务消息
     */
    public void sendRequestMessage(String message) {
        kafkaTemplate.send(requestTopic, message);
        System.out.println("【项目A】发送原始消息：" + message);
    }
}
```

## 3. 消费者：监听 topic-response 接收结果
```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

/**
 * 项目A消费者：接收项目B处理后的返回结果
 */
@Service
public class ProjectAConsumer {

    @Value("${kafka.topic.response}")
    private String responseTopic;

    /**
     * 监听项目B返回的处理结果
     */
    @KafkaListener(topics = "${kafka.topic.response}", groupId = "${spring.kafka.consumer.group-id}")
    public void receiveResponse(String message) {
        System.out.println("【项目A】收到项目B返回结果：" + message);
    }
}
```

## 4. 测试接口（方便手动发送消息）
```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class TestController {
    private final ProjectAProducer producer;

    public TestController(ProjectAProducer producer) {
        this.producer = producer;
    }

    @GetMapping("/send/{msg}")
    public String sendMessage(@PathVariable String msg) {
        producer.sendRequestMessage(msg);
        return "消息已发送到Kafka";
    }
}
```

# 第二套项目：项目B（消费者 + 生产者）
## 1. application.yml 配置
```yaml
spring:
  kafka:
    # 和项目A完全一致：同一套Kafka
    bootstrap-servers: localhost:9092
    # 生产者配置
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
    # 消费者配置（不同消费组，避免冲突）
    consumer:
      group-id: group-B
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: latest

kafka:
  topic:
    request: topic-request
    response: topic-response
```

## 2. 消费者：监听 topic-request 接收原始消息
```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

/**
 * 项目B消费者：接收项目A发送的原始消息
 */
@Service
public class ProjectBConsumer {

    private final ProjectBProducer producer;

    public ProjectBConsumer(ProjectBProducer producer) {
        this.producer = producer;
    }

    /**
     * 监听项目A的请求消息 → 处理 → 发送结果
     */
    @KafkaListener(topics = "${kafka.topic.request}", groupId = "${spring.kafka.consumer.group-id}")
    public void receiveRequest(String message) {
        System.out.println("【项目B】收到项目A原始消息：" + message);
        
        // 模拟业务处理
        String result = handleBusiness(message);
        
        // 发送处理结果给项目A
        producer.sendResponseMessage(result);
    }

    /**
     * 模拟业务处理逻辑
     */
    private String handleBusiness(String originalMsg) {
        return "【处理完成】原始数据：" + originalMsg + " | 处理结果：SUCCESS_" + System.currentTimeMillis();
    }
}
```

## 3. 生产者：发送处理结果到 topic-response
```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

/**
 * 项目B生产者：发送处理结果给项目A
 */
@Service
public class ProjectBProducer {

    @Value("${kafka.topic.response}")
    private String responseTopic;

    private final KafkaTemplate<String, String> kafkaTemplate;

    public ProjectBProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendResponseMessage(String message) {
        kafkaTemplate.send(responseTopic, message);
        System.out.println("【项目B】发送处理结果：" + message);
    }
}
```

## 核心问题解答
### 1. 第二套项目为什么不能只写生产者？
- 只写生产者：**无法接收项目A的消息**
- 必须用**消费者**监听 `topic-request` 拿到原始数据
- 处理完成后，再用**生产者**把结果发回 `topic-response`

### 2. 不同Topic之间隔离吗？
完全隔离：
- Kafka的Topic是逻辑隔离的，消息不会串 Topic
- 两个项目使用不同Topic，互不干扰

### 3. 消费组必须区分吗？
✅ **必须**
- 项目A消费组：`group-A`
- 项目B消费组：`group-B`
如果消费组相同，会导致消息被负载均衡，无法实现双向通信。

## 完整运行流程
1. 启动Kafka服务
2. 启动项目A
3. 启动项目B
4. 访问项目A接口：`http://localhost:8080/send/测试数据123`
5. 控制台输出：
   ```
   【项目A】发送原始消息：测试数据123
   【项目B】收到项目A原始消息：测试数据123
   【项目B】发送处理结果：【处理完成】原始数据：测试数据123 | 处理结果：SUCCESS_xxx
   【项目A】收到项目B返回结果：【处理完成】原始数据：测试数据123 | 处理结果：SUCCESS_xxx
   ```

---

### 总结
1. **双向通信必须两个Topic**：请求Topic + 响应Topic
2. **两个项目都需要 生产者+消费者**：
   - 项目A：发请求、收结果
   - 项目B：收请求、发结果
3. 同一套Kafka环境，Topic天然隔离，安全可靠
4. 消费组必须区分，避免消息冲突
