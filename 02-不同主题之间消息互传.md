以下是基于 Spring Boot 3 + Kafka 的完整示例代码，用于实现两套系统通过同一 Kafka 集群、不同 Topic 进行异步请求-响应交互。

---

## 系统架构说明

- **项目 A**：发送请求消息到 `topic-request`，并消费 `topic-response` 中的响应消息。  
- **项目 B**：消费 `topic-request` 中的请求消息，处理完成后将结果发送到 `topic-response`。

两者使用相同的 Kafka 集群地址，但使用不同的 Topic 进行解耦。

---

## 1. 项目 A（请求发起方）

### 1.1 配置文件 `application.yml`

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
    consumer:
      group-id: project-a-response-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.JsonDeserializer
      properties:
        spring.json.trusted.packages: "*"
    # 定义两个 Topic 名称（也可通过代码常量管理）
    topic:
      request: topic-request
      response: topic-response
```

### 1.2 DTO 定义

```java
package com.example.projecta.dto;

public class Message {
    private String id;
    private String payload;
    // getters and setters
}
```

### 1.3 生产者服务

```java
package com.example.projecta.service;

import com.example.projecta.dto.Message;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

import java.util.UUID;
import java.util.concurrent.CompletableFuture;

@Service
public class RequestProducer {

    @Value("${spring.kafka.topic.request}")
    private String requestTopic;

    private final KafkaTemplate<String, Message> kafkaTemplate;

    public RequestProducer(KafkaTemplate<String, Message> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    /**
     * 发送请求消息，并返回消息ID用于关联响应
     */
    public String sendRequest(String payload) {
        String id = UUID.randomUUID().toString();
        Message message = new Message();
        message.setId(id);
        message.setPayload(payload);
        kafkaTemplate.send(requestTopic, id, message);
        return id;
    }
}
```

### 1.4 响应消费者

```java
package com.example.projecta.service;

import com.example.projecta.dto.Message;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.CompletableFuture;

@Service
public class ResponseConsumer {

    private static final Logger log = LoggerFactory.getLogger(ResponseConsumer.class);

    // 存储异步请求的 Future，key = 消息ID
    private final ConcurrentHashMap<String, CompletableFuture<Message>> pendingRequests = new ConcurrentHashMap<>();

    @Value("${spring.kafka.topic.response}")
    private String responseTopic;

    @KafkaListener(topics = "${spring.kafka.topic.response}", groupId = "${spring.kafka.consumer.group-id}")
    public void consumeResponse(Message response) {
        log.info("Received response: {}", response);
        CompletableFuture<Message> future = pendingRequests.remove(response.getId());
        if (future != null) {
            future.complete(response);
        } else {
            log.warn("No pending request for id: {}", response.getId());
        }
    }

    /**
     * 注册一个等待响应的 Future
     */
    public CompletableFuture<Message> registerFuture(String id) {
        CompletableFuture<Message> future = new CompletableFuture<>();
        pendingRequests.put(id, future);
        return future;
    }
}
```

### 1.5 控制器（触发请求并同步等待响应）

```java
package com.example.projecta.controller;

import com.example.projecta.dto.Message;
import com.example.projecta.service.RequestProducer;
import com.example.projecta.service.ResponseConsumer;
import org.springframework.web.bind.annotation.*;

import java.util.concurrent.CompletableFuture;
import java.util.concurrent.TimeUnit;

@RestController
@RequestMapping("/api")
public class RequestController {

    private final RequestProducer requestProducer;
    private final ResponseConsumer responseConsumer;

    public RequestController(RequestProducer requestProducer, ResponseConsumer responseConsumer) {
        this.requestProducer = requestProducer;
        this.responseConsumer = responseConsumer;
    }

    @PostMapping("/send")
    public String sendAndWait(@RequestBody String payload) throws Exception {
        String id = requestProducer.sendRequest(payload);
        CompletableFuture<Message> future = responseConsumer.registerFuture(id);
        // 阻塞等待响应（生产环境建议设置超时）
        Message response = future.get(10, TimeUnit.SECONDS);
        return "Response: " + response.getPayload();
    }
}
```

## 2. 项目 B（请求处理方）

### 2.1 配置文件 `application.yml`

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
    consumer:
      group-id: project-b-request-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.JsonDeserializer
      properties:
        spring.json.trusted.packages: "*"
    topic:
      request: topic-request
      response: topic-response
```

### 2.2 DTO（与项目 A 相同）

```java
package com.example.projectb.dto;

public class Message {
    private String id;
    private String payload;
    // getters and setters
}
```

### 2.3 请求消费者 + 响应生产者

```java
package com.example.projectb.service;

import com.example.projectb.dto.Message;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class RequestProcessor {

    private static final Logger log = LoggerFactory.getLogger(RequestProcessor.class);

    @Value("${spring.kafka.topic.response}")
    private String responseTopic;

    private final KafkaTemplate<String, Message> kafkaTemplate;

    public RequestProcessor(KafkaTemplate<String, Message> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @KafkaListener(topics = "${spring.kafka.topic.request}", groupId = "${spring.kafka.consumer.group-id}")
    public void processRequest(Message request) {
        log.info("Received request: {}", request);

        // 模拟业务处理（例如：将 payload 转为大写）
        String result = "Processed: " + request.getPayload().toUpperCase();

        // 构造响应消息，保持相同 ID
        Message response = new Message();
        response.setId(request.getId());
        response.setPayload(result);

        // 发送响应到 response-topic
        kafkaTemplate.send(responseTopic, response.getId(), response);
        log.info("Sent response: {}", response);
    }
}
```

## 3. 运行说明

1. 启动 Kafka 服务（例如使用 `docker run -p 9092:9092 apache/kafka` 或本地安装）。
2. 创建两个 Topic（可选，Kafka 会自动创建）：
   ```bash
   kafka-topics --create --topic topic-request --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1
   kafka-topics --create --topic topic-response --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1
   ```
3. 分别启动项目 A 和项目 B（注意端口冲突，可在各自 `application.yml` 中设置 `server.port`，例如 A 用 8080，B 用 8081）。
4. 向项目 A 发送 POST 请求：
   ```bash
   curl -X POST http://localhost:8080/api/send -H "Content-Type: text/plain" -d "hello kafka"
   ```
   将返回 `Response: Processed: HELLO KAFKA`。

## 关键设计点

- **请求-响应关联**：通过消息中的 `id` 字段将请求和响应关联起来，项目 A 使用 `ConcurrentHashMap` 存储待完成的 `CompletableFuture`，消费者收到响应后根据 ID 完成对应的 Future。
- **同步等待**：项目 A 的控制器使用 `future.get()` 阻塞等待，适合简单场景；实际生产可改用异步非阻塞方式（如返回 `DeferredResult` 或 WebSocket）。
- **序列化**：使用 `JsonSerializer` 和 `JsonDeserializer` 自动处理 Java 对象与 JSON 的转换，注意配置 `spring.json.trusted.packages` 避免反序列化安全限制。
- **Topic 配置**：两个项目均通过 `@Value` 读取 Topic 名称，便于统一管理。

---

## 总结

两套系统均包含生产者和消费者角色，它们通过不同的 Topic 实现单向请求和单向响应，形成完整的请求-响应链路。项目 A 负责发起请求并等待结果，项目 B 负责处理请求并返回结果，彼此通过 Kafka 解耦，可独立扩展。



## 是否需要单独的生产者和消费者类？
可以分开：您也可以创建两个独立的类，一个用 @KafkaListener 消费，另一个注入 KafkaTemplate 发送。这样做在职责上更清晰，适合复杂业务。

也可以合并：如上述示例，将两者放在同一个类中，对于简单的请求-响应场景非常方便。

核心原则：项目 B 既需要消费 topic-request，也需要生产 topic-response。无论采用哪种代码组织方式，这两项能力都必须具备。
